# `DD-383` Port the telegram-codex runtime to a Claude Code CLI bridge

## Method

`TDD + BDD + ATDD`

## Problem

`telegram-codex` shells out to the Codex CLI for managed Telegram replies,
live tmux session sharing, version probing, and the Docker E2E lab. To support
Claude Code users we need the same governed runtime targeting the `claude`
binary, with all Codex-specific CLI flags, stream-event shapes, transcript
locations, and image-input handling replaced by the Claude Code CLI contract.

## Claude Code CLI contract used

- interactive resume: `claude --resume <session-id>`
- managed print-mode reply:
  `claude -p "<prompt>" --resume <session-id> --output-format stream-json
  --verbose --dangerously-skip-permissions`
- version probe: `claude --version`
- transcripts: `~/.claude/projects/<encoded-cwd>/<session-id>.jsonl`, with JSONL
  rows shaped `{ "type": "user"|"assistant", "message": { "role", "content":
  [ { "type": "text", "text" } ] } }`
- stream-json events rendered into the Telegram verbose trace:
  `{"type":"system","subtype":"init"}` (Session resumed),
  `{"type":"assistant",...}` text and `tool_use`, `{"type":"user",...}`
  `tool_result`, and `{"type":"result",...}` (Turn completed; carries the final
  `result` reply text)
- images: referenced by local path in the prompt text so Claude opens them with
  its Read tool (no `-i` flag)

## Acceptance

- `skills/telegram-claude` exists as an isolated mini project with its own git
  repository, `.env` (`VERSION=0.01`), MIT `LICENSE`, `Changes`, `README.md`,
  `docs/`, `t/`, and `tickets/`
- runtime module is `lib/Telegram/Claude/Manager.pm`; CLI surface is
  `dashboard telegram-claude.*` with the same command set as `telegram-codex`
- managed replies resume the saved Claude session in print mode with the
  stream-json contract above and stream real Claude events into the Telegram
  verbose trace
- live tmux sharing targets `claude --resume <session-id>` processes and the
  shared transcript is the Claude Code JSONL under `~/.claude/projects/...`
- downloaded photos and image documents are referenced by local path in the
  prompt; no binary-attach claim is made for audio/voice/video/PDF
- `dashboard telegram-claude.start --version` proxies the real
  `claude --version` output without collector side effects
- per-session state under `~/.telegram-claude/<session-id>/`; shared poll roots
  under `~/.telegram-claude/.shared/<sha1(bot-token)>/`
- the Docker noVNC E2E lab installs `@anthropic-ai/claude-code`
- Docker functional tests pass in the shared `perl-test` container
- `lib/Telegram/Claude/Manager.pm` reaches `100%` statement and subroutine
  coverage and `cover_db` is cleaned
- `README.md`, `docs/`, `AGENT.SKILL.md`, `Changes`, and `tickets/` are updated
  and aligned
